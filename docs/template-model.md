# Модель шаблонов и печатных форм

## Сущности
- `Template`: карточка шаблона (название, модуль, формат DOCX/XLSX, статус).
- `TemplateVersion`: версия файла шаблона.
- `GeneratedDocument`: экземпляр, сформированный по карточке записи.

## Жизненный цикл
1. Администратор создает шаблон и загружает файл версии DOCX/XLSX.
2. Система извлекает и анализирует плейсхолдеры из документа автоматически.
3. В карточке версии показываются:
   - все найденные плейсхолдеры;
   - распознанные значения;
   - неизвестные плейсхолдеры;
   - синтаксические ошибки.
4. Администратор активирует проверенную версию.
5. Пользователь в карточке записи нажимает «Сформировать печатную форму».
6. Генератор берет активную версию и контекст записи.
7. Сформированный файл сохраняется в архив (`GeneratedDocument`).

## Поддерживаемый синтаксис
- Простые поля: `[[person.full_name]]`, `[[si.inventory_number]]`.
- Системные поля: `[[lab.full_name]]`, `[[system.today]]`.
- Форматирование дат: `[[format_date:person.hire_date:dd.MM.yyyy]]`.
- Значение по умолчанию: `[[default:person.snils:не указано]]`.
- Логические блоки: `[[if:person.is_active]] ... [[endif]]`.
- Повторяемые блоки: `[[table_start:person.education]] ... [[table_end:person.education]]`.

## Каталог полей
Для каждой формы (`form1`...`form5`) в коде поддерживается каталог допустимых полей.
Неизвестные ключи не блокируют сохранение версии, но помечаются предупреждением до активации.

## MVP генерации Word
- Движок: `docxtpl`.
- Поддержка:
  - одиночных полей;
  - многострочных полей;
  - повторяемых блоков через списки в контексте.
